Overview

Categorical variables in R are represented either by a

  • character
  • factor

A factor is a special character variable that

  • has an underlying numerical representation,
  • a lookup table to find the values, and
  • an inherent ordering.

Characters

Let's create a test data set.

n <- 12
test <- data.frame(y = rnorm(n))
test$char <- rep(c("a","b","A","B"), each=3)

If you look at the type of char, it will be a character (vector). 

str(test$char)

##  chr [1:12] "a" "a" "a" "b" "b" "b" "A" "A" "A" "B" "B" "B"

If you try to run a summary(), you won't get much.

summary(test)

##        y               char          
##  Min.   :-2.1007   Length:12         
##  1st Qu.:-0.5727   Class :character  
##  Median : 0.2515   Mode  :character  
##  Mean   : 0.1213                     
##  3rd Qu.: 1.1621                     
##  Max.   : 1.8698

##    Length     Class      Mode 
##        12 character character

Instead, you want to use the table() function.

table(test$char)

## a A b B 
## 3 3 3 3

Creating factors

We have already seen that you can create factors using the as.factor() function. But sometimes we want a bit more control. In particular, we may be interested in affecting the ordering of the levels of the factor.

To adjust the ordering of the levels of the factor, use the levels argument of the factor() function.

newDiet = factor(case0501$Diet, 
                 levels = c("NP","N/N85","N/R40","N/R50","lopro","R/R50"))
table(newDiet) # compare to table(case0501$Diet)

## newDiet
##    NP N/N85 N/R40 N/R50 lopro R/R50 
##    49    57    60    71    56    56

Adjusting the reference level

case0501$Diet_fact <- relevel(case0501$Diet_fact, ref = "N/N85")
table(case0501$Diet_fact)

## 
## N/N85 lopro N/R40 N/R50    NP R/R50
